# Sales Page Builder - Reference

The structural detail behind the orchestrator. Read whatever section is relevant to the page being built. Not every section is read every time.

---

## 1. Context Clarity (ask if missing)

Before generating, the orchestrator confirms it has:

1. What is the product/service? (specific - what does it do, for whom)
2. What is the single conversion goal? (email capture, call booking, purchase, download)
3. What is the page type? (long-form / short-form / opt-in / VSL / funnel)
4. What is the price point? (free / sub-$100 / $100-$1,000 / above $1,000)
5. What specific result does the buyer get? (outcome, timeframe, mechanism)
6. What proof exists? (testimonials, statistics, case studies, logos)
7. What is the main objection or fear?
8. **Where is the visitor coming from?** (See Section 3)

---

## 2. Page Type Selection

Choose the correct page type before writing a single word. Wrong format kills conversions regardless of copy quality.

| Page Type | Best For | Key Rule |
|---|---|---|
| **Long-form sales** | Mid to high-ticket, complex offers ($200+) | Full narrative, proof-heavy, multiple CTAs. See Section 5. |
| **Short-form sales** | Low-ticket, simple decisions, digital products (sub-$200) | All key info above the fold. Buyer decides in 30-60 sec. See Section 5. |
| **Skool landing page** | Community offers, often combined with VSL | Hybrid: VSL hero + benefits + community proof |
| **Opt-in / Squeeze** | Lead capture, free magnet | Max 2 form fields. Single promise. No navigation. |
| **VSL Page** | Trust-building offers, coaching, programs | Hook question drives to video. CTA after key milestone. |
| **Funnel Page** | One stage in a multi-page sequence | Designed for one moment in the buyer journey. |

---

## 3. Traffic Source Consideration

Where the visitor came from changes how nurtured they need to be on the page. Same offer, different traffic source = different copy.

| Traffic source | Nurturing needed | Page implication |
|---|---|---|
| **Long YouTube content** | Less | They've heard your voice for 20+ min. Skip the long mirror/story. Get to the offer faster. |
| **Existing email list** | Less | They know you. Lead with offer details. |
| **Short-form social** (TikTok / IG / Shorts) | Medium-high | They've seen 30 sec of you. Need the full story + reframe. |
| **Paid ad / cold click** | Maximum | Need everything: stakes, mirror, reframe, proof, offer. |
| **Podcast guest spot** | Medium | They've heard your voice. Need the offer connection + proof. |
| **Word of mouth / referral** | Very low | Third-party trust. Need clear logistics and an easy yes. |

**Rule:** if they're cold, you do all the work. If they're warm, get out of their way.

---

## 4. Above-the-Fold Rules (NON-NEGOTIABLE)

The visitor decides in 3 seconds. Everything above the fold must earn the next scroll.

### The 4-thing test

The hero (headline + subheading + CTA) must immediately tell the reader:

1. **What you do**
2. **Who it's for**
3. **How it makes their life better**
4. **What you want them to do next**

If the reader doesn't get all four within 3 seconds, the hero fails.

### Element rules

| Element | Requirement | Common Mistake |
|---|---|---|
| **Headline** | Scroll-stopping. Filters for the right reader. | Vague, company-centric, "clever" without substance |
| **Subheading** | Adds specificity OR proof. Never restates the headline. | Restating the headline |
| **Key Benefits** | Max 5 bullets. Outcome-led. | Listing features instead of outcomes |
| **Hero Image/Video** | Shows the result or the person achieving it. | Stock photo of strangers |
| **CTA Button** | Distinctive contrasting color. Benefit text. | "Submit", "Order Now", "Click Here" |
| **Navigation** | Remove all nav links from sales/opt-in pages. | Keeping a full site header that leaks traffic |

(See `headline-writing` skill for headline methodology. See `cta-writing` for CTA copy.)

## 5. Long-form vs Short-form: Structural Differences

The single most important decision for any sales page. Same offer, wrong format = no conversion.

### Long-form (for $200+, complex offers)

- **Hero:** headline + subheading + 2+ CTAs + visual proof above the fold + trust elements visible immediately
- **Section #2:** problem/reframe - why the existing approach fails
- **Then:** named offer with feature list, framework or phased roadmap (3-4 phases), detailed "what's included" per component
- **Pricing:** in context with anchors, ROI math, comparison against doing nothing
- **Plus:** founder origin story (once, deeply, mid-to-late page), guarantee with specific terms, "for you / not for you" filter, FAQ with 8-15 questions
- **Proof:** distributed throughout, not clustered. Mix aggregate metric → individual quotes → mid-page case study → final moment. 3+ distinct testimonial moments. Optional video case studies.
- **CTAs:** 8-10+ placements, same button text repeated by design - after every proof moment, after pricing, after guarantee, at the end
- **Tone:** anti-hype qualifiers deliberately ("not quick", "not easy"), permission-granting moments, founder voice throughout

### Short-form (for sub-$200, simple decisions)

- **Hero contains EVERYTHING the buyer needs:** product title (what it is), who it's for, what's included, price, buy CTA
- **Deliberately omitted:** story/mirror/reframe, framework/roadmap, founder story, detailed guarantee section, "for/not for" filter, FAQ (0-3 questions max)
- **Body:** description 1-3 paragraphs MAX, "what's included" single tight bullet list, 1-3 short testimonials OR just star ratings + review count, preview screenshots if applicable
- **Pricing:** shown plainly, no anchoring, no ROI math
- **CTAs:** 1-2 placements (sticky buy button visible always)
- **Decision time:** 30-60 sec. Fits in 1-2 screens of scroll.

### Side-by-side cheat sheet

| Element | Long-form | Short-form |
|---|---|---|
| Page length | 5,000+ words | Under 1,000 words |
| Hero job | Hook + earn the next scroll | Sell the entire offer in the hero |
| Story / reframe | Required (2nd section) | Skipped |
| Framework / roadmap | Required (3-4 phases) | Skipped |
| What's included | Detailed per component | Single bullet list |
| Testimonials | Distributed, 3+ moments | 1-3 max, or star ratings only |
| Pricing | In context with anchors and ROI math | Shown plainly |
| Guarantee | Own section, specific terms | One-line refund policy |
| Founder story | Deep, mid-to-late page | Skipped |
| FAQ | 8-15 questions | 0-3 max |
| CTA frequency | 8-10+ placements | 1-2 (sticky buy button) |
| Decision time | 5-10 minutes | 30-60 seconds |
| Filter (for/not for) | Own section | Implied through framing |

### How to pick

If the buyer needs to be CONVINCED, use long-form. If they just need INFORMATION (already trust you, or price is impulsive), use short-form. Higher price + colder traffic + more abstract = longer.

## 6. Page-Type Formulas

### Squeeze / Opt-in Page

1. Pattern-interrupt question headline
2. Clear value prop (what they get, in plain language)
3. Minimal form (name + email only)
4. CTA congruent with headline ("Send My Free Guide")
5. Single trust element below the form

> Headline: "Want to Close 30% More Sales Calls Without Sounding Pushy?"
> CTA: "Yes - Send Me the Free Script"

**Split sign-up principle:** name + email on page 1, additional info on page 2. Captures leads even from abandoners.

### VSL Page

| Element | Purpose |
|---|---|
| **Hook question** | Stops the scroll. Creates an information gap. |
| **Price anchor** | Descending anchor makes real price feel like a bargain. |
| **Video** | Builds trust + delivers the core message. |
| **CTA below video** | Appears AFTER the key moment, not before. |
| **Trust elements** | Below or adjacent to video. Reduces risk after the pitch. |

### Skool Landing Page

For community-as-offer (Solopreneur OS, Off-Script, etc.):
- Hero with VSL up top (1-3 min)
- Clear "what you get" (community + content + Q&A calls)
- Proof: member count, testimonials, what's already inside
- CTA: "Start free trial" or "Join now"

## 7. CTA Rules

| Rule | What to Do | Never Do |
|---|---|---|
| **Language** | Benefit-focused, in user's voice | "Order Now", "Submit", "Click Here" |
| **Color** | Contrasting color | Same color as page background |
| **Frequency** | Repeat 3+ times on long pages | Show only at very bottom |
| **Friction** | One reduction line below ("No credit card") | Asking for payment before establishing value |

See the `cta-writing` skill for full methodology including natural-insertion patterns for content vs sales-page placement, and the soft-invite + pre-dismissal pattern.

## 8. Trust Elements

Place in three zones: near first CTA, mid-page after body copy, just before final CTA.

| Element | Best Practice | Weak Version |
|---|---|---|
| **Testimonials** | Tier 1: full name, city, specific number. Tier 2 fallback: emotional/transformation quotes. | Generic praise, anonymous |
| **Logos** | Real recognizable companies | Placeholder or obscure brands |
| **Statistics** | Specific user count, outcome data | "Many customers love it" |
| **Trust Badges** | Real SSL, real guarantee badge | Fake badges |
| **Media Mentions** | Real outlets only | Self-published "as seen in" |

See the `testimonial-selection` skill for the full methodology.

## 9. Mobile Optimization

60%+ of landing page traffic is mobile. Build mobile-first.

- Single-column layout throughout
- CTA buttons minimum 44px height
- Images compressed for speed
- No hover-dependent elements
- Test on a real device, not browser resize

---

## 10. Fatal Mistakes

1. **Navigation links on the page** - every link that's not the CTA is a leak.
2. **Generic CTA text** - "Submit", "Click Here" kill conversions.
3. **Key info below the fold** - if the visitor has to scroll to understand the offer, most won't.
4. **More than one conversion goal** - one page, one action.
5. **Too many form fields** - every field beyond name + email cuts completion.
6. **No social proof** - unsupported claim is just a claim.
7. **Price shown before value is established** - anchor value first.
8. **No guarantee on mid/high-ticket (when one exists)** - use only if real.
9. **No mobile optimization** - broken mobile = lost majority.
10. **P.S. section missing** - second most-read element. Don't waste it.

## 11. Anti-AI Strip List with Alternatives

Run before output. Every strip item gets a one-line replacement.

| Strip this | Do this instead |
|---|---|
| "Only 24 hours left! Act now!" | Real urgency only - actual cohort date or price increase. If neither, omit. |
| "$2,000 value, today only $497" | List what's included with "so you can" framing. Drop the value-vs-price math. |
| Fake guarantee badge | Use only if real and specific. If not, omit. |
| "Game-changer / crushing it / transform / unlock potential / supercharge / 10x" | Specific outcome with a number or date instead. |
| "Get Instant Access / Claim Your Spot / Order Now / Submit / Click Here" | Soft invite + what they get: "If you want [thing], join here." (See `cta-writing`.) |
| "Bonus #1, Bonus #2, total value $5,000!" | One clean list of what's included. No bonus pyramid. |
| "As Seen In: [obscure logos]" | Real media mentions only. If none, omit. |
| Generic praise testimonials | Tier 1 specific-number, or Tier 2 emotional/transformation as fallback. (See `testimonial-selection`.) |
| "P.S. - price goes up at midnight!" | If using P.S., make it warm reinforcement. No fake urgency. |
| The long dash character | Use a hyphen (-) instead. Global vault rule. |
| Manipulative emotion ("you'll regret this forever") | Recognition emotion - name the situation, not the threat. (See `emotion-in-copy`.) |
| "We've all been there" (when reader hasn't) | Only use if reader actually has. Specific not generic. |
| Pushy CTAs | Genuine, non-pushy. CTAs invite, never close. |

## 12. Output Format

Keep two things separate.

**The saved sales page file = the page copy ONLY, publish-ready.** It starts at the headline and ends at the sign-off. Do NOT write any of this into the file: a "PAGE TYPE / TRAFFIC SOURCE / GOAL" config block, a "Status" line, "Design Inspo", "Designer Notes", or a "Notes" section, a document title like "<offer> - Sales Page", a "## Copy" wrapper, or an "ABOVE THE FOLD" label. Just the page itself:

- Headline
- Subheading
- Primary CTA + friction reducer
- Body sections, each with its own section header and 2-3 paragraphs (lead / story, the reframe, what's included, proof, who-it-is-for, close)
- Secondary CTAs where they belong
- P.S. only if warm, never urgency
- Sign-off

**The chat reply (not the file)** is where you note the page type + traffic source you chose and any design direction (CTA button color, hero image, mobile priority). Mention those to the user in conversation so the saved page stays clean and ready to paste straight onto a site.
